## Overview
<!-- One paragraph: what this feature does and why it exists. -->
The Manage College feature enables admins to create, edit, and manage detailed college profiles within the platform. 
It provides a structured interface to maintain college metadata, academic offerings, media assets, club associations, 
key contacts, and placement information. This feature ensures a centralized and consistent representation of colleges 
for discovery, analytics, and student engagement.


## Default Behaviors
1. On page load, the first college in the list is auto-selected
2. Default tab selected: College
3. Layout:
  - Left Panel: College list 
  - Top Bar: Search + Filters + Add Button
  - Right Panel: Selected college details

4. College Listing: 
- System shall display list of colleges, Each college card shows: 
     Name, Logo, Location, Establishment Year, Type (Private/Public)

5. Search & Filter: 
- User can search by college name
- Filters available (future extensibility no action needed now):
     Club, Project, Job, Internship, Mentor, College, Company, Event

6. College Selection 
- Clicking a college:
    Loads full details in right panel and Highlights selected item.

7. College Detail View
   - Section A: Basic Info, College Name + Logo, College ID, Affiliated University, Location, Establishment Year, Type
   - Section B: About College > Rich text (collapsed by default) > “See More” expands full content
   - Section C: Brochure > Multiple PDF uploads > Click → opens/downloads file (provide a pdf-viewer component in lib/shared/ui/),
   - Section D: Promotional Videos, Video thumbnails, Play interaction
   - Section E: Campus Gallery, Image grid display
   - Section F: Local Club Chapters, List of associated clubs, Click → navigates to club page (future extensibility no action needed now) (just provide button).
   - Section G: Degrees Offered, Rich text content.
   - Section H: Key Contacts
                Roles: Vice Chancellor, Groom Coordinator, Communication Coordinator, Placement Head, Placement Coordinator,
   - Section I: Placement Highlights, Rich text including: Highest Package, Average Package, Domains, Recruiters, International placements

8.  CREATE / EDIT COLLEGE 
   - Triggered by: “Add” → Create Mode / “Edit” → Edit Mode

## Business Rules & Constraints
1. All Mandatory filled completed the Send to Publish will enable.
2. College Name must be unique.
3. Use FormBuilder for all the fields. 
4. Upload fields will be out off the FormBuilder so that it would be manageable in future.
5. Size of each pdf/images/videos, max 2MB each 
6. Only 1 brochure allowed
7. Use Quill Editor for Rich Text Editor Supports with a common configuration applied form lib/shared/ui so that can be used in other component
8. Use Tailwind css give theme variables and Angular Martial for all the fields as per their filed type.
9. Want use use S3 aws upload i have key and token, Implement it as Domains Driven Design Pattern so that can be use in any where in inn project.
10. clubs, users, degree_programs and program_specializations seed some data.
    - Seed users with the following college contact roles: VICE_CHANCELLOR, PLACEMENT_HEAD, COORDINATOR, PLACEMENT_COORDINATOR, GROOM_COORDINATOR

college_clubs (Mapping)
degree_programs (
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   college_id UUID REFERENCES colleges(id) ON DELETE CASCADE,
   name TEXT NOT NULL, -- B.Tech, M.Tech
   level TEXT, -- UG / PG / DIPLOMA / PHD
   duration_years INT
);
program_specializations (
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
